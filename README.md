# Запуск
Выполнение скриптов протестировано на **Python 3.13** и ОС **Windows 10**

Для запуска скриптов необходимо:
* Создать виртуальное окружение:

`python -m venv <название_окружения>`

* Активировать окружение:

`<название_окружения>\Scripts\activate>`

* Установить необходимые библиотеки:

`pip install -r requirements.txt`

* Запустить скрипт на одном товаре (Часть 1) командой:

`python script.py "<поисковой_запрос>" "<артикул_товара>"`

* Запустить скрипт 3 раза при предопределённых параметрах (Часть 2) командой:

`python run_tests.py`

# Зависимости
Парсер использует библиотеку **Selenium** для имитации работы браузера, драйвер браузера **undetected_chromebrowser**,
который позволяет обходить анти-бот защиту и библиотеку **fake_useragent** для генерации случайных заголовков
*User-Agent* в запросах, чтобы сайту казалось, что при запуске к нему обращаются из разных точек

# Что сработало и не сработало
Изначально я планировал использовать библиотеку **BeautifulSoup** для получения полного кода страниц браузера, но поняв
что Озон подгружает содержимое страниц динамически перешёл на **Selenium**, но после внедрения новой библиотеки я всё
равно не смог зайти на главную страницу из-за анти-бот защиты. Для её обхода я использовал драйвер
**undetected_chromedriver** и генератор случайных *User-Agent* заголовков **fake_useragent**.

Для обхода защиты на всех этапах я старался имитировать действия реального человека с помощью небольших пауз между
действиями, благодаря чему у меня получилось выполнить скрипт 3 раза подряд без блокировки (Часть 2).

Основной проблемой было чтение данных со страницы с поисковой выдачей. Озон не показывает в одном фрейме
(области просмотра) больше 48 карточек товаров, из-за чего простой скролл до заданной высоты страницы не сработал.
Пришлось читать страницу постепенно, сверху вниз по секциям (в 1 секции 12 карточек), но такой подход тоже не дал
результата. В итоге сработал просмотр по текущему фрейму и скролл до последней известной скрипту карточки товара.

# Устойчивость
Я запускал скрипт 3 раза на разных поисковых запросах и артикулах подряд и получил нужные результаты.

Поисковые запросы я выбирал относительно случайно, по тому что видел в рекомендациях на главной странице Озона, а сами
товары выбирал случайно так, чтобы в одном случае товар находился в первых 50 карточках, второй - в карточках 50-100,
а третий - дальше 100, чтобы показать работу возможных сценариев.

Ранее упоминалось, что я использовал специальные библиотеки и имитирование действий пользователя для обхода защиты, но
для долгосрочной работы скрипта, по моему мнению, нужно соблюдать такие правила:
* Пауза между действиями в 0.5-2 секунды
* Задержка перед вводом поискового запроса в 0.5-1 секунду с загрузки страницы
* Ввод запроса по символам с паузами в 50-200 миллисекунд
* Плавный скроллинг небольшими шагами
* Оставаться на странице достаточное количество времени: 5-10 секунд
